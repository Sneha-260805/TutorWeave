\begin{abstract}
\small
EduAgent is a personalized Agentic AI tutoring system for AI/ML education. It classifies learner questions by difficulty using a fine-tuned \texttt{distilbert-base-uncased} model trained on a purpose-built 2,400-sample synthetic dataset, achieving \textbf{97.92\% test accuracy} — a \textbf{+35.8 pp improvement} over a keyword-heuristic baseline. Topic detection uses \texttt{all-MiniLM-L6-v2} dense embeddings via a two-tier alias+cosine pipeline. A \textbf{two-pass personalized RAG} system retrieves level-appropriate examples (Pass~1) and weak-area targeted examples (Pass~2) from the dataset. Four adaptive teaching modes (remedial, clarification, advance, default), a diminishing-returns mastery model, and used-explanation deduplication ensure responses change as the learner progresses. All prompts sent to the LLM are fully documented in this paper.
\end{abstract}

\section{Introduction}

Most LLM-based tutoring systems are stateless: every learner receives the same explanation regardless of prior knowledge, weak areas, or whether the same concept was explained yesterday. EduAgent addresses this by wrapping an LLM in a \textbf{ReAct-style agentic loop}~\cite{react} — Reason (read learner memory), Act (generate adapted explanation), Observe (evaluate understanding) — that drives measurable adaptation across interactions.

\textbf{Key Contributions:}
\begin{itemize}[leftmargin=*,noitemsep,topsep=1pt]
    \item A custom 2,400-sample instructional dataset built with a Gemini-powered quality-filtered pipeline.
    \item A fine-tuned DistilBERT classifier for 3-class difficulty detection (97.92\% accuracy).
    \item Two-pass personalized RAG: Pass~1 by query similarity, Pass~2 by learner's weak concepts.
    \item Four teaching modes selected algorithmically from mastery scores and evaluator output.
    \item A closed-loop: evaluator structured output $\rightarrow$ mastery update $\rightarrow$ next prompt strategy.
    \item All prompts, instruction blocks, and memory hints fully specified.
\end{itemize}

\section{Dataset Construction}

No public labeled dataset exists for AI/ML educational question difficulty. We built a custom \textbf{2,400-sample} dataset using \textbf{Gemini 2.5 Flash} at temperature~0.45 with structured Pydantic output.

\textbf{Taxonomy:} 25 topics $\times$ 4 subtopics $\times$ 3 levels $\times$ 8 samples $= 2{,}400$ (perfectly balanced: 800 per level).

\textbf{Quality Filters (applied to every generated sample):}
\begin{enumerate}[leftmargin=*,noitemsep,topsep=1pt]
    \item \textbf{Length bounds} — answer word count enforced per level (beginner 50--130, intermediate 75--170, advanced 95--230 words).
    \item \textbf{Level leakage} — reject if banned phrases appear (``for a beginner'', ``difficulty level'', etc.).
    \item \textbf{Near-duplicate} — reject if Jaccard~$\geq 0.82$ or SequenceMatcher~$\geq 0.90$ vs. any accepted sample in the same cell.
    \item \textbf{Generic patterns} — reject ``What is X?'' style questions at intermediate/advanced levels.
\end{enumerate}

\textbf{Split:} 80/10/10 stratified by level and topic (train 1920, val 240, test 240).

\section{DistilBERT Difficulty Classifier}
\label{sec:classifier}

\subsection{Architecture and Choice}

\texttt{distilbert-base-uncased}~\cite{distilbert} (66M params, 6 Transformer layers) was chosen over full BERT for 40\% fewer parameters and 60\% faster inference — critical for real-time tutoring. A 3-class classification head is appended on the \texttt{[CLS]} token. Two engineering fixes were required at inference:

\begin{lstlisting}[language=Python]
# Fix 1: AutoTokenizer avoids BertTokenizer path error
tokenizer = AutoTokenizer.from_pretrained(CLASSIFIER_PATH)
model     = AutoModelForSequenceClassification.from_pretrained(
                CLASSIFIER_PATH)
# Fix 2: DistilBERT has no segment embeddings
inputs = tokenizer(question, return_tensors="pt",
         max_length=128, truncation=True, padding="max_length")
inputs.pop("token_type_ids", None)  # removes TypeError
logits = model(**inputs).logits
level  = LABELS[torch.argmax(logits).item()]
\end{lstlisting}

\subsection{Shared Sentence-Transformer Singleton}

\texttt{ml/embedder.py} loads \texttt{all-MiniLM-L6-v2} (384-dim, 22M params) once at import and exports \texttt{embed\_model}. Both \texttt{retriever.py} and \texttt{topic\_detector.py} import the same Python object — verified by \texttt{id()} equality. This avoids loading 90\,MB twice.

\textbf{Tier~2 — Semantic cosine:} Each topic is encoded as \texttt{"\{topic name\}: \{all its questions concatenated\}"} into a 384-dim vector. Topic vectors are cached per \texttt{(dataset, level)} pair. At query time only the user question is encoded; $O(1)$ model calls per query regardless of dataset size.

\textbf{Complexity penalty:} Each retrieved example is penalized by up to 0.30 before ranking: +0.15 if answer $>$18 words; +0.15 if any hard term (``derive'', ``convergence'', ``subgradient'', etc.) appears. This prevents advanced examples from being retrieved for beginner queries despite high surface similarity.

\textbf{Pass-2 query:} For each weak concept (e.g., ``chain rule''), the query is \texttt{f"\{concept\} \{topic\}"} — encoding both the concept and its context. Results across all concepts are pooled, deduplicated against Pass-1, and the top-2 returned.

\subsection{Mastery Model with Diminishing Returns}

The mastery score $m \in [0, 1]$ per topic is initialized at \textbf{0.5} (maximum-entropy prior) and updated after every evaluation:

\begin{align}
\text{good:}\quad & m \leftarrow \min(1.0,\; m + 0.20 \times (1-m)) \\
\text{partial:}\quad & m \leftarrow \max(0.0,\; m - 0.06) \\
\text{poor:}\quad & m \leftarrow \max(0.0,\; m - 0.18)
\end{align}

The factor $(1-m)$ in Rule~(1) guarantees diminishing returns: gain at $m{=}0.5$ is $0.10$; at $m{=}0.9$ it is only $0.02$. This prevents artificial saturation after a few correct answers.

\subsection{Used-Explanation Tracking}

After every tutor response, the tag \texttt{f"\{level\}-\{mode\}"} (e.g., \texttt{"intermediate-clarification"}) is appended (once, deduplicated) to \texttt{profile["used\_explanations"][topic]}. The memory hint then instructs the LLM:
\begin{lstlisting}
Avoid repeating the exact same prior explanation style.
Previously used: beginner-default, intermediate-remedial.
\end{lstlisting}

\subsection{Mode-Specific Instruction Blocks}
\label{sec:mode_prompts}

The \texttt{\{mode\_specific\_instruction\}} placeholder is replaced with one of the following four verbatim blocks, selected by \texttt{build\_mode\_specific\_instruction(teaching\_mode)}.

\subsubsection{Remedial Mode}
\textit{Trigger: evaluator=``poor'' OR (mastery~$<$~0.45 AND weak areas/count~$\geq$~3)}

\begin{lstlisting}
Mode-specific instructions:
- Re-teach from scratch.
- Use very simple wording.
- Use one small concrete example.
- Explain in 4 to 6 short sentences.
- Focus on one main idea first before adding detail.
- Avoid abstract definitions unless absolutely necessary.
- Avoid repeating the same explanation style or analogy
  used earlier for this topic.
- Start with: "Let's simplify it completely."
\end{lstlisting}

\subsubsection{Clarification Mode}
\textit{Trigger: evaluator=``partial'' OR (weak areas AND mastery~$\leq$~0.75)}

\begin{lstlisting}
Mode-specific instructions:
- Briefly restate the main idea in one or two simple sentences.
- Then focus on the weak or confusing concept.
- Use one clear example.
- Do not repeat the whole long explanation.
- Keep the answer focused and moderately short.
\end{lstlisting}

\subsubsection{Advance Mode}
\textit{Trigger: evaluator=``good'' OR (mastery~$>$~0.75 AND topic count~$\geq$~2)}

\begin{lstlisting}
Mode-specific instructions:
- Assume the learner understood the basics.
- Avoid repeating the full beginner introduction.
- Give a slightly deeper explanation.
- Connect the concept to a related next-step idea.
- Keep the answer clear but more intellectually rich.
\end{lstlisting}

\subsubsection{Default Mode}
\textit{Trigger: no evaluator hint AND no strong mastery signal}

\begin{lstlisting}
Mode-specific instructions:
- Give a normal level-appropriate explanation.
\end{lstlisting}

\label{sec:eval_fq}

Sent at \textbf{temperature=0.2} (slight variety across sessions, still focused).

\begin{lstlisting}
You are an evaluator for an adaptive AI tutor.

Topic: {topic}
Learner level: {level}

Original learner question:
{user_question}

{tutor_answer}

\subsection{Evaluator Agent — Reply Evaluation Prompt}
\label{sec:eval_reply}

Sent at \textbf{temperature=0.0} (fully deterministic — evaluation drives mastery updates).

\begin{lstlisting}
You are an evaluator for an adaptive AI tutor.

Topic: {topic}
Learner level: {level}

Follow-up question:
{followup_question}

Learner reply:
{learner_reply}

Evaluate the learner's reply.

Return valid JSON only with exactly these keys:
- understanding_level: one of ["good", "partial", "poor"]
- weak_concepts: list of short concept names
- feedback: short educational feedback
- recommended_action: one of ["advance", "re-explain",
   "give easier example", "give more practice"]

Rules:
- Be fair and educational.
- If the learner is partly correct, mark as "partial".
- Keep weak_concepts short and specific.
- Do not include markdown.
- Return valid JSON only.
\end{lstlisting}

\textbf{Expected output:}
\begin{lstlisting}
{
  "understanding_level": "partial",
  "weak_concepts": ["chain rule", "gradient flow"],
  "feedback": "You identified backpropagation's purpose but
               missed the chain rule's role in computing
               layer-wise gradients.",
  "recommended_action": "give more practice"
}
\end{lstlisting}

Output is validated by Pydantic \texttt{EvaluationResult} before touching the profile. On any parse failure, safe fallback is returned: \texttt{understanding\_level="partial"} (not ``poor'', to avoid incorrectly triggering remedial mode).

\subsection{Memory Agent — Hint Strings (Verbatim)}
\label{sec:memory_hints}

The Memory Agent builds natural-language strings injected into the tutor prompt. No LLM call — pure profile logic.

\subsubsection{build\_memory\_hint — All Possible Lines}

\begin{lstlisting}
# topic_count > 1
"The learner has seen this topic before. Do not restart
with the exact same beginner introduction."

# weak_areas[topic] non-empty
"The learner has previously struggled with: {concepts}.
Address these concepts carefully."

# mastery < 0.40
"The learner appears weak in this topic. Use simpler
wording, intuition, and one concrete example."

# mastery > 0.75
"The learner appears comfortable with this topic.
You may go slightly deeper than before."

# used_explanations[topic] non-empty
"Avoid repeating the exact same prior explanation style
if possible. Previously used: {tags}."

# NONE of the above (first visit, clean state)
"No strong prior history for this topic.
Start with a clear explanation."
\end{lstlisting}

\subsubsection{build\_evaluation\_strategy\_hint — All Possible Strings}

Only fires when \texttt{last\_evaluation["topic"] == current topic} (same-topic guard).

\begin{lstlisting}
# understanding_level == "poor"
"Teaching mode: remedial. The learner did not understand
the previous explanation. Do NOT repeat the same
analogy-driven explanation. Re-teach from scratch using
very simple wording, one tiny concrete example, and a
short step-by-step explanation."

# understanding_level == "partial"
"Teaching mode: clarification. The learner understood
partially. Do not repeat the whole answer. Briefly
restate the core idea, then focus on the confusing part
with one clear example."

# understanding_level == "good"
"Teaching mode: advance. The learner understood the
previous explanation well. Do not repeat the full
beginner introduction. Give a slightly deeper explanation
and connect it to the next related concept."
\end{lstlisting}

Action addenda appended based on \texttt{recommended\_action}:
\begin{lstlisting}
# "give easier example"
"Use one very easy real-world or numeric example."
# "re-explain"
"Explain from scratch in the simplest possible way."
# "give more practice"
"After explaining, add one short practice-style check."
# "advance"
"After the explanation, connect to the next topic."
\end{lstlisting}

If \texttt{weak\_concepts} non-empty, also inserts:
\begin{lstlisting}
"Focus mainly on these weak concepts first: {concepts}."
\end{lstlisting}

\section{Infrastructure}

\subsection{LLM Client (Retry + Fallback)}

\texttt{agents/llm\_client.py} wraps the Groq API with retry and graceful fallback:

\begin{lstlisting}[language=Python]
attempts = max(1, LLM_MAX_RETRIES + 1)  # default: 3
for attempt in range(attempts):
    try:
        response = client.chat.completions.create(
            model=model, messages=messages,
            temperature=temperature,
            timeout=LLM_TIMEOUT_SECONDS)  # 8s
        content = response.choices[0].message.content
        if content and content.strip():
            return content.strip()
    except Exception:
        if attempt < attempts - 1:
            time.sleep(0.75 * (attempt + 1))  # 0.75s, 1.5s
if fallback is not None:
    return fallback
raise RuntimeError("All retries failed.")
\end{lstlisting}

\subsection{Database and Security}

\textbf{SQLite WAL mode:} \texttt{PRAGMA journal\_mode=WAL} allows concurrent reads during writes. \texttt{PRAGMA synchronous=NORMAL} balances durability and speed. Complex profile fields (mastery, weak\_areas, etc.) stored as JSON strings, loaded with \texttt{\_safe\_json\_load()} that returns empty defaults on NULL or malformed data.

\textbf{Password security:} PBKDF2-SHA256 with 200,000 iterations (NIST SP~800-132), 16-byte \texttt{os.urandom()} salt, stored as base64(salt~$\|$~hash). Verification uses \texttt{hmac.compare\_digest} for timing-attack resistance.

\section{Limitations and Future Work}

\textbf{Limitations:}
\begin{itemize}[leftmargin=*,noitemsep,topsep=1pt]
    \item Classifier trained on synthetic data only; may not generalize to informal real-world phrasing with typos.
    \item Heuristic mastery constants not calibrated on real learner data.
    \item Weak concepts accumulate indefinitely; no automatic forgetting.
    \item No long-context transcript memory across turns.
\end{itemize}

\textbf{Future Work:} Replace heuristic mastery with Bayesian Knowledge Tracing~\cite{bkt}; add persistent FAISS vector store; implement weak-concept decay; extend to multi-domain STEM topics; use RL to learn optimal mode-selection policy.

% ==========================================================
\section{Conclusion}

EduAgent demonstrates that a practical personalized tutoring system can be built by combining a fine-tuned difficulty classifier, semantic retrieval, and a feedback-driven mastery model. The DistilBERT classifier achieves \textbf{97.92\% accuracy} (+35.8 pp over the heuristic baseline) on a purpose-built dataset. The two-pass RAG, four teaching modes, and evaluation-driven strategy hints create a closed adaptive loop where every component's output modifies the next answer. All agent prompts, instruction blocks, and memory hint strings are fully documented in this paper, making the system fully reproducible and auditable.
